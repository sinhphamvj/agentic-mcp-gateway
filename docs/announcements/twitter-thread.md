🔌 The Problem: 
The N-to-N integration between various LLMs and Model Context Protocol (MCP) tool servers is a nightmare. Routing intents, standardizing inputs, and managing context across different models takes too much time.

**Tweet 3:**
🛠️ The Solution:
A unified gateway architecture! 
User Request ➡️ Intent Classifier (LangGraph) ➡️ Agents ➡️ Specific MCP Servers.
Everything is configured via a simple `workflow.yaml`. No hardcoded API keys, no messy integrations.

**Tweet 4:**
📊 Built for Production:
Observability is built-in. Every LLM generation and MCP tool call is traced with OpenTelemetry and Arize-Phoenix. Plus, we support human-in-the-loop for destructive operations via LangGraph interrupts. 🔍
